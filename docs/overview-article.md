# 4SLAS CMS — бесплатная система управления блогом для каждого

*Июнь 2026*

**4SLAS** — это полностью бесплатная, самодостаточная CMS для блога, написанная на чистом PHP 8. Создана для всех, кто хочет запустить свой сайт быстро, без лишних заморочек и ежемесячных платежей.

Никаких фреймворков, никаких подписок, никаких «облаков» за ваши деньги. Просто код, который ты полностью контролируешь.

Разберём, что под капотом.

## Что внутри

### База

**Стек:** PHP 8 + MySQL + Vanilla JavaScript.  
**21 таблица** в базе данных. **26 страниц админки**. **2 языка** (русский и английский). **~12 000 строк PHP**.

Конфигурация через `.env` — никаких захардоженных паролей (да, мы это специально проверили и починили). Установщик из пяти шагов.

### Безопасность — всерьёз

Провели полный аудит (66 находок, все критические закрыты). Вот что работает из коробки:

- **CSRF-токены** на всех формах — никто не подменит запрос
- **Пароли** — Argon2ID, самый стойкий алгоритм хеширования в PHP
- **Защита от брутфорса** — 5 попыток входа за 15 минут с IP
- **Капча** — текстовая + графическая (GD)
- **Honeypot** — скрытое поле-ловушка для ботов
- **Защита сессий** — httponly, strict_mode, samesite, secure
- **SVG-санитайз** — вырезаем скрипты и обработчики событий
- **Подготовленные запросы** — 100% PDO, ни одной SQL-инъекции
- **Защита от перебора пользователей** — одинаковая ошибка для всех случаев
- **Блокировка установщика** после инсталляции

### Вход — как удобно

Три способа авторизоваться:

1. **Классика:** логин + email + пароль → подтверждение почты
2. **Яндекс ID:** OAuth 2.0, один клик, авто-регистрация
3. **VK ID:** OAuth 2.0, один клик, авто-регистрация

OAuth-провайдеры настраиваются в админке независимо. Кириллические имена транслитерируются автоматически. Кнопки «Войти через...» встроены везде: на странице входа, на странице регистрации, в плавающей панели.

### Редактор — свой, не чужой

Собственный WYSIWYG-редактор `4SLASeditor` на 1383 строки JavaScript. Что умеет:

- всё форматирование текста (жирный, курсив, заголовки, цитаты...)
- загрузка картинок drag-and-drop
- вставка видео (YouTube, VK, Rutube)
- вставка таблиц
- вставка кода с подсветкой
- полноэкранный режим
- просмотр и редактирование HTML-кода
- счётчик слов и символов

Иконки в тулбаре — чистые SVG. Никаких эмодзи, всё как в серьёзных редакторах.

> **Хочешь drag-and-drop?** Есть отдельный аддон — GrapesJS Upgrade. Визуальный конструктор с блочной палитрой и панелью стилей. Устанавливается за минуту через админку. [Подробнее →](#drag-and-drop-аддон-grapesjs-upgrade)

## Drag-and-Drop аддон (GrapesJS Upgrade)

Это отдельный пакет расширения — `dgr_update`. Он превращает 4SLAS из текстового блога в полноценный визуальный конструктор. Устанавливается опционально, не затрагивая базу данных безвозвратно (всё в транзакции).

### Что даёт аддон

**1. Визуальный конструктор GrapesJS**
В админке поста и страницы появляется переключатель:
```
[ Визуальный конструктор ]  [ Текстовый редактор ]
```
- Блочная палитра слева: текст, картинка, видео, кнопка, карта, секция, колонки
- Панель стилей справа: шрифты, отступы, тени, flex, анимации
- Drag-and-drop блоков прямо на канвасе
- Адаптивный предпросмотр (десктоп / планшет / телефон)
- Контент синхронизирован между редакторами — переключайся без потерь

**2. Умная работа с картинками**
При создании черновика через GrapesJS картинки уходят во временную папку `uploads/temp/editor/`. При первой публикации поста они автоматически переносятся в постоянную папку и конвертируются в WebP.

**3. Drag-and-drop менеджер блоков**
Кастомные HTML-блоки (реклама, виджеты, баннеры) теперь можно:
- Перетаскивать для сортировки внутри позиции (мышкой, как папки)
- Перемещать между позициями (`leftmenu` ↔ `footer` ↔ `after_first_post`)
- Создавать новые позиции на лету

**4. Динамические позиции**
Вместо трёх жёстко заданных позиций — система динамических позиций. `renderBlocks('твоя_позиция')` в любом шаблоне — и блоки отображаются в порядке `sort_order`.

### Как установить

**Способ 1: через админку (рекомендуется)**

1. Зайди в `admin/update.php` → блок «Обновление CMS»
2. Заархивируй папку `dgr_update/` в ZIP (из корня проекта: `zip -r dgr_update.zip dgr_update/`)
3. Загрузи ZIP-архив через форму загрузки
4. Система сама: распакует → проверит содержимое → сделает бэкап → заменит файлы → выполнит миграцию БД → очистит кеш
5. Готово — в админке постов появится тумблер «Визуальный конструктор»

**Способ 2: через консоль**

```bash
cd /путь/к/сайту
php dgr_update/4SUPDATE.php
```

**Способ 3: через браузер (требует авторизацию админа)**

Открой `https://вашсайт/dgr_update/4SUPDATE.php?token=...` (доступ только из админки через `admin/update.php`)

### Что произойдёт при установке

| Этап | Действие |
|---|---|
| Копирование vendor | 4 файла GrapesJS + SortableJS → `src/vendor/` (~1.2 МБ) |
| Новый файл | `src/grapesjs-init.js` — инициализация конструктора |
| Бэкап | 9 файлов CMS копируются в `/backup/pre_update_ДАТА/` |
| Замена | 9 файлов CMS обновляются (редакторы, блоки, API, шаблоны) |
| Миграция БД | `custom_blocks.sort_order` + настройка `available_positions` (всё в транзакции) |
| Сброс кеша | OPcache + файловый кеш |

### Как откатить

Если что-то пошло не так:

```bash
# 1. Вернуть файлы из бэкапа
cp -r backup/pre_update_ДАТА/* .

# 2. Откатить БД
mysql -u user -p dbname -e "ALTER TABLE custom_blocks DROP COLUMN sort_order; DELETE FROM settings WHERE setting_key='available_positions';"
```

После отката — очисти кеш в админке, и сайт вернётся в состояние «до».

### Безопасность

Аддон прошёл полный аудит (28 находок, все критические закрыты):
- Галерея: whitelist `jpg/jpeg/png/gif/webp` (защита от RCE)
- Картинки: `basename()` защита от path traversal при миграции
- API загрузки: CSRF-проверка
- Миграция БД: в транзакции с rollback при сбое
- Бэкап: `.htaccess` блокирует прямой доступ к `/backup/`
- Защита от прямого веб-запуска `4SUPDATE.php` без авторизации
